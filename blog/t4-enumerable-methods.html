How To Use The .group_by Method
11/18/2014

The enumerable module provides a number of methods that are used to traverse, sort, arrange and rearrange collections. These methods have proven EXTREMELY useful as I've made my way through Phase 0. The method I'm going to teach is a method called .group_by.

The thing to remember about enumerables, is that they are used on collections. The .group_by method in its most simplistic use can take a collection and pass it into a hash that describes the way you wanted the original collection grouped. This will be much easier to understand through examples. Let's say you have a collection of numbers from 1 to 30 and you want to group it by number left over when divided by 4. In programming this is called the modulus and it is represented with the % sign. So for instance 5%4 returns 1, because the remainder of 5/4 is 1. Therefore, if we go back to our example above we can see that our collection of numbers will be broken down into 4 groups. The group that has a remainder of 0, 1, 2, and 3 when divided by 4. This is what the code would look like:

(1..30).group_by {|i| i%4}

It would return a hash like this:

{0=>[1,2,3,4,8,12,16,20,24,28], 1 => [5,9,13,17,21,25,29], 2 => [6,10,14,18,22,26,30], 3 => [7,11,15,19,23,27]}

Ok, so maybe you're starting to get it but that doesn't seem like a relevant use of grouping. What if you had a collection of book titles and you wanted to group them by genre. You would have to use a bit more complex code by the idea would be the same:

books = ["book1", "book2", "book3", "book4"] 

genre_hash = books.group_by do |genre|
   case rating
    when "book1" || "book2"
        "historical fiction"
    when "book3"
        "science fiction"
    when "book4"
        "non-fiction"
    end
end

This would return a hash like this:

{"historical fiction"=>["book1", "book2"], "science fiction"=>["book3"], "non-fiction"=>[book4]}

Hope these examples helped you understand the group_by method!
