Learning New Languages

Loops in JavaScript

12/09/2014 by Flori

This week at DBC has been all about learning a new language, JavaScript. It is most commonly used as a part of a web browser. It allows your code to interact with the user and it can alter the content that is being displayed on a webpage. In a lot of ways knowing the language Ruby has helped me learn JavaScript. I know what variables and loops are and I am starting to understand the power of programming languages. However, Ruby and JavaScript are very different. Sometimes knowing one, complicates the way you "speak" in another. It reminds me of the time I took Italian in college. I was already fluent in Spanish and that made it pretty easy for me to read Italian, but when I tried to speak Italian I was often confusing the two languages with each other. So what's so different about these two languages, you might ask.

Ruby is considered one of the most user friendly programming languages out there. Even to someone new to coding, Ruby programs can be understood because they have an easy to follow syntax. This is similar to how some languages are easy to read because they sound how they look (they're phonetic). JavaScript is not as friendly a language as Ruby. It favors communicating with the computer over communicating with the programmer. For instance, let's look at loops in Ruby vs. JavaScript.

Ruby has a number of methods that make looping easy. If you want to do something 6 times, you use the .times method:


6.times {puts "Hello")
=> Hello
Hello
Hello
Hello
Hello
Hello

See what I mean? Even if you didn't know anything about programming you could probably figure out what that line of code was about to do.

Ruby also has enumerable methods that you can use for every element in a collection. The beauty of this is, you don't care how many elements are in this collection, you just want to do it for each one of those elements. For example,


number_array = [1,2,3,4,5,6]
number_array.each {|num| puts num}
=>1
2
3
4
5
6

Again, as someone completely new to programming, other than learning what puts does, it is pretty easy to guess what the code above will do. Ruby also has nifty keywords like "unless" so you can do something unless some condition holds. All these types of loops in JavaScript are covered by a for loop. The syntax for that is:


for (var i = 0; i<n; i++) {
  //block of code
}

The for loop takes three arguments in JavaScript. The first one (var i = 0) initializes a variable, the second one (i<n) says to repeat the loop while i <n, the last one (i++) reassigns the value of i after each loop by adding one (i++ is fancy for saying i = i+1). Here is an example:


var alphabet_array = ["a","b","c","d","e","f"]

for (var i = 0; i<alphabet_array.length; i++){
  console.log(alphabet_array[i])
}

=>a
b
c
d
e
f

So as you can see by using array.length we are basically adding the same functionality as the .each enumerable in Ruby. While JavaScript can provide the same functionality, its version of this loop is a little harder to understand at first glance, and there is more specific rules about syntax so it's easier to forget something. Ruby and JavaScript do have one loop in common, the while loop. In Ruby the loop looks like this:


i=0
while i<number_array.length
  puts ("number at position #{i} = #{number_array[0]}")
  i++
end

In Java, the same code looks like this:


var i = 0;

while (i <n= alphabet_array.length) {
  console.log ("letter at position " + i + " is " + alphabet_array[i]);
  i++;
}

These loops certainly resemble each other but again you can see the small subtleties in their syntaxes. JavaScript is a powerful language but as far as learning it goes, it will be a little harder to grasp that power because it is not as user friendly as Ruby.
