Classes in Ruby

and Cars

11/25/2014

This week was all about creating and understanding classes in Ruby. I'm going to attempt to pass off some of my recently acquired knowledge on this topic to you.

Classes are extremely important when it comes to structuring code. They allow a developer to model real life situations. Inside of a class live numerous instance variables and methods. Once a class is created, an instance of that class can be created. This instance and any other instance of the class that may be created will have access to those method. That's probably a little confusing so let's take a look at a couple of examples.

First we have class String, which is a built in ruby class and thus, when we create an instance of it, that instance has access to every method inside the class String. Ex:

apple = "this is an instance of the class string"
#we have created an instance of the Ruby class String.
#Inside of String there is a method called reverse that reverses the string. Therefore, we can call this method on any instance of string, including apple.
apple.reverse = "gnirts ssalc eht fo ecnatsni na si siht"

Above is an example of a built in class, but Ruby allows you to create your own classes and create unique methods that are only available to instances of that class. Let's see what this looks like:

#creating a class called Classes
            class Classes

            attr_reader :students
#creating a method to initialize the class requiring a class name and the number of student in the class
                def initialize(title,students)

                    @title = title

                    @students = students

                end
#Creating a method that let's you add a student or students to an instance of class Classes
                def add_students(number)

                    @students += number

                end

            end
#creates an instance of the class classes with 23 students 
        algebra = Classes.new(algebra, 23)

#adds five student to the algebra instance of Classes
        algebra.add_students(5)

#should return the number of students in the class which should now be 28
        algebra.students

I know, that was kind of mean...a class called Classes. Maybe this will help shed some light. You could have a class called Car that is initialized with a make, model and color. Then you can create numerous instances of the car. For example, my first car was a honda civic that went by the name Delila. Therefore, I would create an instance of the class Car by writing: 

        Delila = Car.new(Honda, Civic 2008, black)

Delila would then have access to any of the methods that I created within the class Car. So if the class Car had a get_a_paint_job method, I could call this method: Delila.get_a_paint_job(red)
and then when I ask for Delila's color:
Delila.color
it will return red. (This is a little bit simplistic and ignoring the fact that class Car would need a read method for color, but that is out of the scope of this blog. I just want you to understand how classes allow instances of that class to access their methods and how powerful these methods can be.)

This week we have really dived deep into understanding classes, their variables and their methods. Classes are powerful tools that allow developers to structure their code in a logical manner. They are the building blocks to readable, organized and creative coding. Hopefully, I was able to give you a little sense of how useful classes can be.
